** Specification

Given a small team; maybe augmented by AI; then how to contribute collectively to the
project?

** Fork and pull request
:PROPERTIES:
:ID:       c8f29915-b70c-4ca7-b0de-ecd01406112b
:END:

1. Fork the repository.
2. git remote add upstream $UPSTREAM_URL
3. Define or choose a task in the [[ref:90abac79-d226-4de2-9174-9df6c48c25c0][backlog]].
4. Assign yourself to the task.
5. Add/update/delete relevant [[ref:1f6eeb40-6e6b-4e19-9d5d-23af3858023d][specifications]] and implementations.
6. Send a [[ref:5deb7507-29b1-44da-9286-a81ced5f594c][PR]].
7. Synchronize with upstream
   1. git fetch upstream
   2. git checkout master
   3. git rebase upstream/master
   4. git push origin master

** Ruleset

The [[ref:c8f29915-b70c-4ca7-b0de-ecd01406112b][Fork and pull request]] is enforced by a rule set:
- https://github.com/org-00000/infra/settings/rules/16035411

** Pull request
:PROPERTIES:
:ID:       5deb7507-29b1-44da-9286-a81ced5f594c
:END:

Before sending a PR, check that:
- [ ] The code compiles cleanly.
- [ ] The code is formatted.
- [ ] All tests pass.
- [ ] Relevant code conventions are satisfied.
- [ ] Documentation has been updated.
- [ ] Your task is marked as =DONE=.
