* Architecture
:PROPERTIES:
:ID: 94a78f42-9098-4796-90c1-725e20a07fbd
:END:

The architecture describes the [[ref:5c5ca183-a932-4744-b483-a5c419755639][clocks]] software. clocks exists to provide services
needed by the [[ref:e1e52c89-31a6-4394-ad4c-ba8951dd80b3][Timerly]] business case. This description expresses /what/ clocks does and
/how/ it does it.

** Extending the architecture

The architecture can be viewed as a graph of specifications. To extend the
architecture is to remove, update or add specifications and/or how they are related.

See the [[ref:a9be14e1-29bd-4858-9ac9-87a6480d6f34][Specification]] section to learn how to build a specification. Update the
[[ref:94a78f42-9098-4796-90c1-725e20a07fbd][Architecture]] section to update the architecture.

** clocks
:PROPERTIES:
:ID:       5c5ca183-a932-4744-b483-a5c419755639
:END:

We call /clocks/ the unix process started with /make start/. clocks listens for HTTP
requests on localhost:4000 and satisfies this [[ref:9dff3c23-619d-45e5-be70-a00410129737][protocol]].

1. [[ref:66e3c137-9013-46c3-b153-991434a69ad9][make process]] starts this [[ref:b47fa002-6ace-46bd-8821-1fdd5838d939][application]]
2. which starts this [[ref:d229871a-95ee-4a49-8a40-44332fa8977d][endpoint]]
3. which configures a [[ref:eb404cfc-7d96-4a09-bc36-d27597e937f1][Bandit]] web server
4. which listens to HTTP traffic from localhost:4000 as [[ref:1228a3b7-6b81-4698-b046-cf657657d079][configured]] in [[ref:d4a51e4e-d5a1-403c-940e-07f520fc531c][dev mode]].
5. which replies to HTTP queries using a [[ref:b95d3d95-cbf1-442c-bf52-0b049373a9fe][router]] that [[ref:30bce120-ef3c-45df-a736-efaf76bddbfe][satisfies]] the protocol.

** protocol
:PROPERTIES:
:ID:       9dff3c23-619d-45e5-be70-a00410129737
:END:

- GET /api/health ↦ 200 { "health": "ok" }

- GET / ↦ 302 /v0

- GET /v0 ↦ 200 [[ref:db3920ac-d997-43a2-8e91-d3f83541034a][Root]]

- POST /v0 pdf ↦
  1. | 400 /v0 if pdf is not a PDF.
  2. | 500 /v0 if a [[ref:e91dd873-5e94-412c-96b9-ae6d04976bb4][clock]] fails to start from pdf.
  3. | 303 /v0/clock/id where id is the id of a clock built from
     the pdf.

- GET /v0/clock/id ↦
  1. | a [[ref:1401a5d0-1e6f-49d7-a789-836d93c7b0b1][ClockLiveview]] for the matching [[ref:e91dd873-5e94-412c-96b9-ae6d04976bb4][clock]] if any.
  2. | 200 /v0 otherwise.

- ? _ ! 404 /v0

** Root
:PROPERTIES:

** Clock
:PROPERTIES:
:ID:       e91dd873-5e94-412c-96b9-ae6d04976bb4
:END:

Description
- A Clock is an [[ref:94ec1e6c-fdcd-4737-8876-54c61137ac6f][actor]] that emits a [[ref:309fd88e-4408-4ea8-b221-e2d36ffc9170][signal]] every second that contains the
  number of elapsed seconds for as long as it has been alive.

Constructors
- clock : [[ref:0975ffbf-0375-4760-ac27-2ab30b8d21a7][Pdf]] → [[ref:5dd2a34c-6f34-4b56-b9a7-449fcc7ca453][Result]](Clock)
  - For a given PDF, there is at most one Clock.

- clock : [[ref:6c22b7a9-40f5-4340-a966-bd099f7ebeae][Id]] → Result(Clock)

Interface
- field : Clock → Result(String)
- id : Clock → Id

Signals
- [[ref:309fd88e-4408-4ea8-b221-e2d36ffc9170][Signal]]

A Clock is supervised.
