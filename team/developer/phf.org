** Bridge
*** 1. Understanding the Bridge Architecture

Without a bridge, your VM traffic goes through NAT — the VM has a
private IP invisible to the rest of your network. With a bridge, the
VM is a direct participant on your LAN and receives its own IP from
your router's DHCP, just like any physical machine.

**** Network topology with br0

| Component    | Role                                                                 |
|--------------+----------------------------------------------------------------------|
| Physical NIC | e.g. =enp0s31f6= — your Ethernet card, enslaved to br0              |
| br0          | The bridge: acts as a virtual switch. Gets the IP from DHCP.        |
| vnetX        | Virtual NIC created by libvirt for each VM, plugged into br0        |
| VM           | Sees its own virtual NIC, gets an IP from your router directly      |

The key point: once =enp0s31f6= is enslaved to =br0=, the physical NIC
no longer holds an IP address. The bridge =br0= takes over that role.

#+BEGIN_QUOTE
*Note:* A bridge over a WiFi interface (=wlp*=, =wlan*=) is not
supported by most drivers. You must use a wired Ethernet interface
for =br0=.
#+END_QUOTE

*** 2. Prerequisites

Install the required packages:

#+BEGIN_SRC bash
sudo apt update
sudo apt install \
    qemu-kvm \
    libvirt-daemon-system \
    libvirt-clients \
    bridge-utils \
    virt-manager \
    virtinst \
    qemu-utils \
    cpu-checker \
    virt-viewer \
    spice-vdagent \
    gir1.2-spiceclientgtk-3.0
#+END_SRC

Add your user to the required groups (log out and back in afterwards):

#+BEGIN_SRC bash
sudo usermod -aG kvm,libvirt $USER
#+END_SRC

*** 3. Identify Your Network Interface

Before creating a bridge, find the name of your active Ethernet interface:

#+BEGIN_SRC bash
ip addr show
#+END_SRC

Look for an interface that is =UP= and has an =inet= address:

#+BEGIN_SRC
2: enp0s31f6: <BROADCAST,MULTICAST,UP,LOWER_UP> mtu 1500 ...
    link/ether 48:2a:e3:0f:69:8f brd ff:ff:ff:ff:ff:ff
    inet 192.168.1.100/24 metric 1024 brd 192.168.1.255 ...
#+END_SRC

| Flag / Field       | Meaning                                                              |
|--------------------+----------------------------------------------------------------------|
| =<...UP,LOWER_UP>= | Interface is administratively UP and has physical link (cable in)   |
| =<NO-CARRIER,...>= | No physical link — cable unplugged. Bridge will not work.           |
| =inet 192.168.1.x= | Current IP — the bridge will take this over                         |
| =link/ether=       | MAC address of the physical NIC                                     |

*** 4. Create the Bridge with NetworkManager

**** Step 1 — Create the bridge interface

#+BEGIN_SRC bash
sudo nmcli connection add type bridge \
    ifname br0 \
    con-name br0 \
    bridge.stp no
#+END_SRC

=bridge.stp no= disables Spanning Tree Protocol, which adds a 30-second
delay at startup. Safe to disable in a home/lab setup with no loops.

**** Step 2 — Slave your Ethernet NIC to the bridge

Replace =enp0s31f6= with your actual interface name:

#+BEGIN_SRC bash
sudo nmcli connection add type ethernet \
    ifname enp0s31f6 \
    con-name br0-slave \
    master br0
#+END_SRC

**** Step 3 — Disable the old Ethernet connection

#+BEGIN_SRC bash
# List all connections
nmcli connection show

# Bring down the old Ethernet connection (replace name with yours)
sudo nmcli connection down 'Wired connection 1'
sudo nmcli connection modify 'Wired connection 1' connection.autoconnect no
#+END_SRC

**** Step 4 — Bring up the bridge

#+BEGIN_SRC bash
sudo nmcli connection up br0
sudo nmcli connection up br0-slave
#+END_SRC

*** 5. Verify the Bridge

**** Check interfaces and IPs

#+BEGIN_SRC bash
ip addr show
#+END_SRC

What you should see:
- =enp0s31f6= has *no* =inet= address (it has been enslaved to =br0=)
- =br0= has the =inet= address assigned by your router
- =br0= state is =UP=

#+BEGIN_SRC
2: enp0s31f6: <BROADCAST,MULTICAST,UP,LOWER_UP> mtu 1500 ... master br0
    link/ether 48:2a:e3:0f:69:8f ...
    # no inet line — correct!

4: br0: <BROADCAST,MULTICAST,UP,LOWER_UP> mtu 1500 ...
    link/ether 8e:dc:58:7d:ed:65 ...
    inet 192.168.1.14/24 metric 1024 brd 192.168.1.255 ...
#+END_SRC

**** Check bridge member ports

#+BEGIN_SRC bash
bridge link show
#+END_SRC

Expected output:

#+BEGIN_SRC
2: enp0s31f6: <BROADCAST,MULTICAST,UP,LOWER_UP> mtu 1500 master br0 state forwarding ...
9: vnet3: <BROADCAST,MULTICAST,UP,LOWER_UP> mtu 1500 master br0 state forwarding ...
#+END_SRC

| Value             | Meaning                                              |
|-------------------+------------------------------------------------------|
| =state forwarding= | Port is healthy and passing traffic — correct       |
| =state disabled=  | No carrier (cable unplugged) or STP blocking        |
| =master br0=      | This interface is enslaved to br0                   |
| =vnetX=           | libvirt auto-creates this NIC when a VM starts      |

**** Check routing

#+BEGIN_SRC bash
ip route
#+END_SRC

You should see a default route via =br0=:

#+BEGIN_SRC
default via 192.168.1.1 dev br0 proto dhcp src 192.168.1.14 metric 1024
192.168.1.0/24 dev br0 proto kernel scope link src 192.168.1.14
#+END_SRC

#+BEGIN_QUOTE
***Note:* If you see two default routes (one via =wlp4s0= and one via
=br0=), WiFi is also active. This is harmless — the VM uses =br0= and
your host routes normally.
#+END_QUOTE

*** 6. Configure the VM to Use br0

Edit the VM XML with =virsh edit <vmname>= and set the interface block:

#+BEGIN_SRC xml
<interface type="bridge">
  <mac address="52:54:00:xx:xx:xx"/>
  <source bridge="br0"/>
  <model type="virtio"/>
</interface>
#+END_SRC

*** 7. Attach an ISO and Set Boot Order

**** CDROM block with ISO attached

#+BEGIN_SRC xml
<disk type="file" device="cdrom">
  <driver name="qemu" type="raw"/>
  <source file="/home/user/iso/Fedora-Workstation-Live-43-1.6.x86_64.iso"/>
  <target dev="sda" bus="sata"/>
  <readonly/>
  <address type="drive" controller="0" bus="0" target="0" unit="0"/>
</disk>
#+END_SRC

**** Boot order (cdrom before hd)

#+BEGIN_SRC xml
<os firmware="efi">
  <type arch="x86_64" machine="pc-q35-7.2">hvm</type>
  <boot dev="cdrom"/>
  <boot dev="hd"/>
</os>
#+END_SRC

#+BEGIN_QUOTE
*Note:* After installation, remove the =<boot dev="cdrom"/>= line and
remove the =<source>= element from the cdrom disk block so the VM boots
from disk on next start.
#+END_QUOTE

*** 8. Troubleshooting

| Symptom                                        | Fix                                              |
|------------------------------------------------+--------------------------------------------------|
| =bridge link show= shows =disabled= on NIC     | Cable not plugged in — br0 has no uplink         |
| =br0= has no =inet= address                    | Run: =sudo dhclient br0=                         |
| "Activation of network connection failed"      | Check the above two; also verify libvirt uses br0|
| EFI shell instead of ISO boot                  | No =<source>= in CDROM block, or wrong boot order|
| virt-manager SPICE console error               | =sudo apt install gir1.2-spiceclientgtk-3.0=     |

*** 9. Quick Reference Checklist

1. Install packages (section 2)
2. Identify Ethernet interface: =ip addr show=
3. Create =br0= and slave Ethernet NIC: =nmcli= (section 4)
4. Verify: =ip addr show=, =bridge link show=, =ip route=
5. Configure VM XML: =type="bridge"=, ISO attached, =cdrom= boot first
6. Start VM and confirm it gets an IP from your router
